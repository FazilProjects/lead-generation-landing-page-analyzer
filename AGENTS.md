# Agent Instructions

## Project Positioning

Keep this project honest and portfolio-ready.

This is a local JavaScript landing page audit dashboard for Fazil Waseem. Do not present it as a real AI platform, production CRO product, analytics tool, Google Ads tool, Meta Ads tool, or live landing page analyzer.

## Technical Rules

- Use HTML, CSS, and vanilla JavaScript.
- Do not add external APIs.
- Do not fetch or analyze live URLs.
- Do not add ad platform integrations.
- Do not add analytics or tracking scripts.
- Keep GitHub Pages compatibility.
- Keep logic readable and local.

## UI Rules

- Maintain a polished dashboard feel.
- Keep the layout responsive at desktop, tablet, and mobile widths.
- Prevent horizontal overflow.
- Keep buttons easy to tap on small screens.
- Use accessible labels, focus styles, and semantic HTML.
- Keep status labels visible as text, not color only.

## Content Rules

- Keep recommendations practical and client-style.
- Mention that output is generated from local rule-based planning logic.
- Include portfolio disclaimers where appropriate.
- Avoid claims about real AI, live performance prediction, crawling, tracking, or platform data.

## Scoring Rules

Preserve simple scoring behavior:

- Vague or short offers reduce offer clarity.
- Vague CTAs such as Submit or Learn More reduce CTA strength.
- Too many lead form fields reduce the form friction score.
- Trust-related words improve trust proof.
- Google Ads recommendations emphasize search intent and message match.
- Meta Ads recommendations emphasize hook, proof, offer clarity, and mobile experience.
- Organic recommendations emphasize SEO clarity, trust, and content flow.
- Mixed traffic recommendations emphasize message consistency and segmented CTA paths.
